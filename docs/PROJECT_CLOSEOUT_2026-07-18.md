| 源码、文档、示例配置 | 是 |

| 项 | 内容 |
|----|------|
| 管理页面 | https://cash.weini.xin |
| 短信接口 | `POST https://cash.weini.xin/api/v1/sms` |
| 部署目录 | `/home/ubuntu/cashpulse/` |
| 运行方式 | Docker Compose + 外部网络 `webproxy` |
| 反代 | 主机 Caddy 增加 `cash.weini.xin` → `cashpulse:8080` |
| 数据 | `/home/ubuntu/cashpulse/data/cashpulse.db`（实例私有） |
| 资源 | 空闲约 **6–7 MiB** 内存，CPU 近 0 |

### 「永不熄灭」是什么意思

容器策略为 **`restart: unless-stopped`**：

- 进程异常退出 → Docker **会自动拉起**  
- 主机重启后（Docker 服务正常时）→ 容器 **会再起来**  
- 只有人为 `docker compose stop` / `docker rm` / 改策略，才会停住  

所以对日常使用而言：**服务按设计是常驻、可自愈的**，不是「关了电脑就没了」的本地临时进程。

> 注意：这不等于绝对 100% SLA。磁盘满、主机宕机、手动 stop、镜像/二进制被删等仍会导致不可用，需要运维层面关注。

## 5. 使用方式备忘（不含密钥正文）

1. **网页**：打开管理域名 → 输入 **管理员密码** 登录（密码仅存服务器 `.env`，不写进本文）。  
2. **iPhone 快捷指令**：收到银行短信 → POST 全文到 `/api/v1/sms`，Header 带 **INGEST_TOKEN**。  
3. 密钥位置：服务器 `/home/ubuntu/cashpulse/.env`（权限应保持私有）。
